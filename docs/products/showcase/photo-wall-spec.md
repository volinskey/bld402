# Showcase: Photo Wall

Live demo at **wall.run402.com** — a curated gallery of fun AI-generated images.

## Source Template

`templates/utility/photo-wall/` (schema.sql, index.html, rls.json)

The showcase is built FROM this template with demo-specific modifications listed below.

## Functional Requirements

### FR-1: Gallery Grid

- Displays all photos in a responsive masonry-style grid.
- Each photo shows: image, caption, and upload date.
- Grid adapts from 4 columns (desktop) to 2 columns (tablet) to 1 column (mobile).
- All 12-15 AI-generated images are visible on page load.

### FR-2: Lightbox

- Clicking any photo opens a full-screen lightbox overlay.
- Lightbox shows the full-size image with caption below.
- Left/right arrow keys and swipe gestures navigate between photos.
- Close button (X) or clicking outside the image dismisses the lightbox.
- ESC key also dismisses the lightbox.

### FR-3: Captions

- Each photo has a short, playful caption displayed below the image in the grid.
- Captions are also shown in the lightbox view.

### FR-4: Uploads Disabled (demo-specific)

- The upload bar / upload button from the template is completely removed from the HTML.
- No file input, drag-and-drop zone, or upload UI is present.
- The showcase is strictly view-only.

### FR-5: Auth Removed (demo-specific)

- Auth modal and sign-in button are removed from the HTML.
- No authentication UI is shown since uploads are disabled.

### FR-6: Seed Photos (demo-specific)

- 12-15 pre-loaded AI-generated images with playful captions:
  1. "When the code finally compiles"
  2. "Office dog says hi"
  3. "Abstract art by a robot"
  4. "Sunset generated by AI"
  5. "404: Sleep not found"
  6. "The cloud is just someone else's computer"
  7. "Debugging mood"
  8. "When you fix one bug and create three more"
  9. "Monday morning vibes"
  10. "The rubber duck knows all"
  11. "Deploy on Friday? Sure, what could go wrong"
  12. "Stack overflow to the rescue"
- All photos have `is_seed = true` in the database.
- No delete or edit UI is present.

### FR-7: Demo Notice Banner

- Yellow banner at top: "This is a curated demo wall. Build your own with bld402!"

### FR-8: Footer

- "Built with bld402 · Powered by run402" with links.

## Schema

Based on template `schema.sql` with demo-specific additions:

```sql
CREATE TABLE photos (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  image_url text NOT NULL,
  caption text,
  user_id uuid,
  is_seed boolean DEFAULT false,
  created_at timestamptz DEFAULT now()
);

CREATE INDEX idx_photos_created ON photos(created_at DESC);
```

**Differences from template:**
- Added `is_seed boolean DEFAULT false` — marks demo photos

## RLS

Based on template `rls.json` with demo-specific modifications:

- **SELECT:** anon can read all rows
- **INSERT:** disabled (uploads removed in showcase)
- **UPDATE:** disabled
- **DELETE:** disabled

```sql
CREATE POLICY "anon_select" ON photos FOR SELECT TO anon
  USING (true);
```

No INSERT, UPDATE, or DELETE policies are needed for the showcase since uploads are disabled.

## Template Repeatability

**The key value of each showcase app is proving that the template works.** Anyone following the bld402 steps with this template MUST be able to reach the same functional result. The only differences between a fresh build and the showcase are the pinned demo modifications listed above. A fresh build will include the upload bar and auth UI that are removed in the showcase.

**Red Team must validate both:**
1. **The live demo** at wall.run402.com — test all acceptance criteria above.
2. **A fresh build from scratch** — follow bld402.com steps using `templates/utility/photo-wall/`, provision a new project, deploy, and verify the template produces a working app with the same core functionality (photo grid, lightbox, upload, captions).
