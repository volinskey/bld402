MCP — Frequently Asked Questions

Quick answers about bld402-mcp.

What does bld402-mcp do?

It gives your AI assistant the ability to build and deploy web apps for you. You describe what you want, your assistant does the rest.

How much does it cost?

Nothing to try. It uses free test money (testnet USDC). If you want to keep your app running long-term, plans start at $5/month.

Is it safe?

Yes. It only creates web apps on run402 servers. It cannot access your local files, send emails, or do anything outside of building your app. See the safety page for details.

What can it build?

Web apps with databases, user accounts, file uploads, serverless functions, and more. 13 templates included (todo lists, games, blogs, voting apps) or build custom apps from scratch.

What AI assistants work with it?

Claude Code, Cursor, and any tool that supports MCP (Model Context Protocol). You can also use the bld402 command-line tool directly.

How do I install it?

Add one JSON block to your AI assistant's MCP settings. See the install guide.

What happens after 7 days?

Prototype apps go read-only for 7 days, then get archived. Upgrade to keep running ($5/month). Your coding agent can handle the upgrade for you.

Can other people use my app?

Yes! You get a shareable link (like myapp.run402.com) that works on any device. No sign-up needed for your users.

How is this different from the website workflow?

The website workflow guides your agent step-by-step through bld402.com. MCP is faster — your agent gets direct tool access and can build an app in a single call, without visiting any website.

Can I use both?

Yes. They produce the same result — a live web app on run402. MCP is just a faster path if your assistant supports it.
